1. namespace engines - хранит информацию о двигателях некоторого типа
	в частности при реализации для двигателей внутреннего сгорания организована структура InternalCombustionEngine
	для добавления новых типов двигателей создать аналогичную структуру

2. namespace teststand - для тестирования двигателей по различным показателям
	в частности при реализации для определения времени перегрева организован класс TestOverheatTime
	для добавления тестов нового типа создать аналогичный класс
	класс шаблонный для охвата всех видов двигателей
	
3. namespace interfaceio - для взаимодействия с консолью (ввод-вывод)
	нет классов и структур, только функции для взаимодействия
	
4. значения кусочно-линейной функции в каждый следующий момент времени (дельта t = 1) рассчитывается через коэффициенты подобия:
		зная координаты правого и левого конца отрезка можно посчитать значение функции внутри него
		
5. к температуре двигателя при пересчете "+" как нагрев, так и охлаждение,
								т.к. по формуле охлаждения значение всегда отрицательное
								
6. так как крутящий момент (по графику) убывает до нуля, то при определенных температурах внешней среды перегрев двигателя не наступает
	для таких ситуаций введена переменная ограничивающая время (MAX_TIME)
	если же изменить входные (крутящий момент постоянный или увеличивается) то в этом члучае таких ситуаций не возникает
